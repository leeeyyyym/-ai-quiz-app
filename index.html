Chat, Project, Skills, Artifacts, Cowork를 이해하고, 수업용 HTML 웹앱을 만든 뒤 실제 수업에 맞게 수정·점검합니다.

  • Cowork로 파일 기반 작업 실행

2차시 · 배포와 공유

GitHub Pages로 웹앱 배포하기

1차시에서 만든 HTML 웹앱을 index.html로 준비하고 GitHub Pages에 올려, 학생에게 공유할 수 있는 접속 주소를 만듭니다.

  • 로컬 실행과 웹 배포 차이 이해
  • GitHub 저장소 만들기

2차시 안내서 열기

1차시에서 2차시로 이어지는 흐름

1차시 결과물이 2차시의 출발점입니다. 1차시가 “만들기”라면, 2차시는 “공유 가능한 주소 만들기”입니다.

Step 1

Artifacts로 HTML 웹앱 만들기

교과 개념 퀴즈, 자기점검표, 자료 해석 도구, 루브릭 안내 도구 중 하나를 만듭니다.

Step 2

수업용 v2로 고도화하기

글씨와 버튼을 키우고, 학생 안내문을 넣고, 교사가 수정할 수 있도록 주석과 데이터 구조를 정리합니다.

Step 3

배포 전 안전 점검하기

학생 개인정보, 외부 API, 저작권 자료, 공식 평가 오해 가능성을 확인합니다.

Step 4

파일명을 index.html로 바꾸기

GitHub Pages에서 첫 화면으로 열리도록 최종 HTML 파일명을 index.html로 준비합니다.

Step 5

GitHub Pages로 배포하기

저장소에 파일을 올리고 Pages를 켜서 학생에게 공유할 수 있는 URL을 만듭니다.

두 차시 전체 운영안

구분 핵심 질문 주요 활동 산출물
1차시 AI에게 어떤 수업 도구를 만들게 할 것인가? Project·Skills·Artifacts·Cowork 이해, HTML 웹앱 제작과 고도화 수업용 HTML 웹앱 v2
2차시 만든 웹앱을 어떻게 학생에게 공유할 것인가? GitHub 저장소 생성, index.html 업로드, Pages 배포 공유 가능한 GitHub Pages URL
핵심 연결 문장 1차시의 최종 파일이 2차시의 시작 파일입니다. 1차시에서 완성한 v2 HTML을 index.html로 바꾸면 GitHub Pages 배포 실습으로 자연스럽게 이어집니다.

2차시로 넘어가기 전 점검

아래 항목을 확인한 뒤 GitHub Pages에 배포합니다.

배포 전 주의 GitHub Pages에 올리면 로컬 파일이 아니라 인터넷 주소가 있는 공개 자료가 됩니다. 학생 개인정보나 내부 업무 자료가 포함된 파일은 배포하지 않습니다.

연결용 프롬프트

1차시 결과물을 2차시 배포용으로 정리할 때 아래 프롬프트를 사용합니다.

프롬프트 — 2차시 배포용 HTML 정리 
이 HTML 웹앱을 GitHub Pages에 배포하기 좋은 최종본으로 정리해줘.

조건:
- 파일명은 index.html로 사용할 예정
- HTML, CSS, JavaScript가 한 파일 안에서 작동
- 학생 개인정보 입력칸 없음
- 외부 API 호출 없음
- API 키나 비밀값 없음
- 학생용 안내문 포함
- 결과는 공식 평가가 아니라 학습 확인 또는 자기점검용이라는 안내 포함
- 모바일과 태블릿에서도 보기 좋게 반응형 적용
- 교사가 나중에 수정할 부분에는 주석 추가
- 배포 전 교사가 확인해야 할 체크리스트를 마지막에 주석으로 넣어줘

최종 목표

이 연수의 최종 목표는 교사가 AI로 수업 도구를 만들고, 수업 목적에 맞게 고도화한 뒤, 개인정보와 저작권을 점검하고, 안전하게 공유 가능한 웹 주소까지 만드는 것입니다.
