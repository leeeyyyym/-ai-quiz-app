0. 최신 업데이트 — 지금 무엇이 달라졌나

생성형 AI는 단순히 질문에 답하는 도구에서, 코딩하고 자료를 다루고 여러 단계의 일을 처리하는 작업 수행형 AI로 빠르게 바뀌고 있습니다. 오늘 연수의 핵심도 바로 이 흐름을 수업과 업무에 어떻게 연결할 것인지입니다.

GPT 계열 흐름

ChatGPT에서 사용하는 GPT 계열 모델은 계속 더 빠르고, 더 정교하고, 더 많은 작업을 처리하는 방향으로 발전하고 있습니다. 최근에는 GPT-5.6 계열인 Sol, Terra, Luna가 제한 프리뷰로 공개되어 API와 Codex 일부 환경에서 먼저 제공되고 있습니다.

다만 중요한 점은, GPT-5.6은 아직 일반 ChatGPT 사용자에게 제공되는 단계가 아니라는 것입니다. 오늘은 모델 이름을 외우기보다, AI가 점점 더 세분화되고 특히 코딩과 작업 수행 능력이 강화되고 있다는 흐름을 이해하면 됩니다.

Claude 계열 흐름

Claude는 Opus, Sonnet, Haiku처럼 성능과 속도에 따라 모델군을 나누어 설명해 왔습니다. 최근에는 Fable 5, Mythos 5처럼 더 새로운 세대의 모델도 등장했습니다.

이 역시 이름 자체보다 중요한 것은 방향입니다. Claude도 대화형 답변을 넘어서, 문서·코드·파일·작업 흐름을 다루는 기능이 강화되고 있습니다.

오늘 기억할 핵심 GPT와 Claude 모두 관심의 초점을 “대화를 잘하는 AI”에서 “코딩하고, 자료를 다루고, 여러 단계의 일을 처리하는 AI”로 옮기고 있습니다.
수강생 안내 모델 이름과 버전은 계속 바뀝니다. 오늘 연수에서는 최신 모델명을 외우는 것보다 Chat → Artifacts → Cowork로 갈수록 AI에게 맡기는 작업 범위와 권한이 커진다는 점을 이해하는 데 집중합니다.

1. 오늘 연수 흐름

AI 사용은 Chat → 데스크톱 앱 → CLI로 갈수록 권한과 자율성이 커집니다. 오늘은 Chat 기능을 중심으로 실습하고, 마지막에 Cowork의 파일 기반 작업을 살펴봅니다.

Project·Skills

3. 실습 1 — Project와 Skills로 반복 업무 정리하기

첫 번째 실습은 가정통신문처럼 반복되는 문서 작업을 작업방(Project)과 업무 매뉴얼(Skills)로 정리하는 것입니다.

실습 1-1. Project 이름 정하기

예시 Project 이름 학교 안내문·가정통신문 작성 프로젝트

실습 1-2. 가정통신문 작성 규칙 정리하기

프롬프트 1 — 작성 규칙 정리 
가정통신문 작성용 Skill을 만들기 위해 필요한 규칙을 정리해줘.

학교에서 학부모에게 보내는 안내문 형식이고, 정중하고 따뜻한 존댓말을 사용해야 해.
제목, 인사말, 주요 내용, 일정, 준비물, 문의처, 마무리 문구가 포함되면 좋겠어.

다음 항목으로 정리해줘.
1. 사용할 상황
2. 문서 구조
3. 문체와 어조
4. 반드시 포함할 항목
5. 피해야 할 표현
6. 예시 문장
7. 최종 점검 기준

실습 1-3. Skill 지침 형태로 정리하기

프롬프트 2 — SKILL.md 형태로 정리 
위 내용을 바탕으로 가정통신문 작성 Skill의 지침을 만들어줘.

SKILL.md에 들어갈 수 있도록 다음 구조로 정리해줘.
- 목적
- 사용 상황
- 작성 원칙
- 문서 구조

- 점검 기준

학교 현장에서 바로 재사용할 수 있도록 너무 추상적이지 않게 작성해줘.
핵심 정리 반복되는 업무는 매번 새로 설명하지 말고, 규칙으로 만들어 재사용할 수 있습니다.

4. 실습 2 — Artifacts로 수업용 웹앱 만들기

이번 실습은 “Agent 실습”이 아니라 Artifacts 실습입니다. 우리가 원하는 수업 도구를 설명하면 Claude가 결과물을 만들고, 우리는 그 결과를 보면서 계속 수정합니다.

공통 실습 — 교과 개념 확인 퀴즈 웹앱

프롬프트 3 — 교과 개념 확인 퀴즈 웹앱 
고등학교 수업에서 사용할 교과 개념 확인 퀴즈 웹앱을 만들어줘.

요구사항은 다음과 같아.
- 객관식 5문항
- 학생이 답을 선택하면 정답 여부 표시
- 각 문항마다 간단한 해설 제공
- 마지막에 총점 표시
- 오답 문항을 다시 확인할 수 있게 구성
- 학생 이름이나 개인정보는 입력받지 않음
- 교실 전자칠판에서 보기 좋게 글씨를 크게
- HTML 한 파일로 만들어줘

주제는 [교과 개념]으로 해줘.

수정 요청 예시

난이도·내용 수정

  • 문제 난이도를 고등학교 2학년 수준으로 높여줘.
  • 해설을 더 자세하게 바꿔줘.
  • 오답을 선택했을 때 왜 틀렸는지 설명해줘.
  • 문항 순서를 무작위로 섞어줘.

기능·화면 수정

  • 정답 확인 버튼을 따로 만들어줘.
  • 점수에 따라 피드백 문구가 다르게 나오게 해줘.
  • 전자칠판에서 보기 좋게 글씨와 버튼을 크게 해줘.
  • 교사가 문제를 쉽게 수정할 수 있도록 코드에 주석을 달아줘.

선택 실습 A — 데이터 분석 수행평가 자기점검표

프롬프트 4 — 자기점검표 웹앱 
데이터 분석 수행평가용 자기점검표 웹앱을 하나의 HTML 파일로 만들어줘.

기본 설정:
- 대상: 학생 본인 점검용
- 형태: 항목을 탭하면 체크되는 인터랙티브 웹앱
- 입력 내용은 자동 저장(localStorage)
- 상단에서 학생이 과목·평가명·이름(학번)을 직접 입력

점검 항목은 아래 흐름 순서대로 4단계로 묶어줘.

1. 주제 정하기
- 수집할 데이터의 주제를 두 가지 이상 정했다
- 한 가지면 감점이라는 붉은 경고 표시

2. 데이터 다루기
- 기본 제공 양식에 데이터를 작성했다
- 두 데이터 각각을 정리하고 열 통계를 확인했다
- 또는, 함수를 활용해 데이터를 처리했다
- “또는” 관계임을 시각적으로 표시

3. 시각화하기
- 두 데이터를 차트로 시각화했다
- 그 차트를 선택한 이유를 작성했다

4. 해석하기
- 두 결과를 서로 연관지어 살펴봤다
- 연관지어 시사점을 도출했다

기능:
- 완료한 항목 비율을 원형 진행률로 시각화
- 하단에 성찰 서술란: 시사점 / 차트 선택 이유 / 보완하고 싶은 점
- 채운 내용을 텍스트 파일(.txt)로 다운로드하는 버튼
- 한글이 깨지지 않게 처리
- 전체 초기화 버튼

디자인:
- 점검·기록 도구다운 차분하고 정돈된 느낌
- 데이터 분석의 단계적 흐름이 드러나는 번호형 구조
- 모바일에서도 잘 보이게 반응형으로

선택 실습 B — 자료 해석 연습 도구

프롬프트 5 — 자료 해석 연습 도구 
고등학교 [교과] 수업에서 사용할 자료 해석 연습 웹앱을 만들어줘.

요구사항:
- 짧은 자료 또는 제시문을 보여줌
- 학생이 핵심 내용을 선택하거나 답을 고를 수 있음
- 선택 후 해설 제공
- 자료 해석 포인트를 단계적으로 안내
- 학생 이름이나 개인정보는 입력받지 않음
- HTML 한 파일로 제작

선택 실습 C — 발표·토론 루브릭 안내 도구

프롬프트 6 — 루브릭 안내 도구 
고등학교 발표 수업에서 사용할 루브릭 안내 웹앱을 만들어줘.

요구사항:
- 발표 평가 기준을 영역별로 보여줌
- 내용, 근거, 표현, 시간 준수, 질의응답 항목 포함
- 학생이 자기 발표 준비 상태를 체크할 수 있음
- 체크 결과에 따라 보완할 점 안내
- 점수 산출이 아니라 자기점검용으로 구성
- 학생 이름이나 개인정보는 입력받지 않음
- HTML 한 파일로 제작

5. Artifacts로 만든 웹앱의 실행 범위

Artifacts에서 만든 웹앱은 기본적으로 인터넷에 자동 배포되는 서비스가 아닙니다. Claude 안에서 미리 보거나, HTML 파일로 저장해서 내 컴퓨터에서 여는 수업 도구입니다.

중요 학생들이 각자 휴대폰이나 노트북으로 접속하는 사이트를 만들려면 별도의 웹 배포 과정이 필요합니다. 오늘 연수에서는 배포가 아니라 수업 도구 설계와 검토에 집중합니다.
구분 의미 오늘 실습과의 관계
Artifacts 미리보기 Claude 화면 안에서 결과물을 바로 확인하는 상태 오늘 직접 확인
HTML 파일 실행 내 컴퓨터에서 파일을 열어 사용하는 상태 가능
로컬 실행 내 브라우저에서만 실행되는 상태 오늘 기본 방식
웹 배포 인터넷 주소를 만들어 다른 사람도 접속하게 공개하는 상태 오늘은 다루지 않음
로컬 실행이어도 주의할 점 웹앱 안에 외부 API 호출, 구글시트 연동, 로그인, 저장 기능이 들어가면 외부 서버와 통신할 수 있습니다. 학생 이름, 성적, 상담 내용 같은 개인정보를 입력받는 구조는 피해야 합니다.

6. Claude Cowork 앱으로 작업 실행하기

Cowork는 단순히 답변을 받는 채팅이 아니라, Claude에게 파일 기반 작업을 맡기는 환경입니다. 파일을 읽고, 검토하고, 수정본을 만드는 흐름을 안전하게 경험합니다.

Chat과 Cowork의 차이

구분 Chat Cowork
기본 성격 질문하고 답변 받기 작업을 맡기기
주요 용도 초안 작성, 설명, 아이디어 파일 읽기, 수정, 정리, 장시간 작업
작업 방식 한 번 답변 중심 계획 → 실행 → 결과 확인
주의점 답변 검토 권한 범위와 파일 변경 여부 확인

실습용 폴더 준비

폴더 이름 예시 AI연수_실습자료

이 폴더에는 실제 학생 자료가 아니라 가상 자료만 넣습니다.

넣어도 되는 예시

  • 가상_퀴즈앱.html
  • 수행평가_체크리스트_예시.html
  • 가정통신문_가상초안.txt
  • 연수_메모.txt

넣으면 안 되는 예시

  • 실제 학생 명단
  • 실제 성적 자료
  • 상담 기록
  • 건강 정보·생활지도 기록

AI연수_실습자료 폴더에 있는 HTML 퀴즈앱 파일을 검토해줘.

검토 기준은 다음과 같아.
- 버튼이 정상 작동하는지
- 정답과 점수 계산 구조가 명확한지
- 교사가 나중에 문항을 쉽게 수정할 수 있는지
- 학생 이름이나 개인정보를 입력받는 부분이 있는지
- 전자칠판에서 보기 좋게 글씨가 충분히 큰지

아직 파일을 수정하지 말고, 먼저 개선이 필요한 부분만 목록으로 알려줘.

프롬프트 9 — 원본 보호 후 수정본 저장 
방금 검토한 내용을 바탕으로 HTML 퀴즈앱을 수정해줘.

수정 조건은 다음과 같아.
- 원본 파일은 그대로 두기
- 수정본은 파일명 뒤에 _수정본을 붙여 새 파일로 저장하기
- 교사가 문항을 쉽게 바꿀 수 있도록 문제 데이터 부분에 주석 달기
- 개인정보 입력칸은 만들지 않기
- 전자칠판용으로 글씨와 버튼을 크게 하기

7. 개인정보와 학교 현장 적용 유의사항

AI는 초안을 만드는 도구이지, 교사의 판단을 대신하는 도구가 아닙니다. 수업용 웹앱이나 문서 작성 도구를 만들 때 개인정보와 평가 공정성을 반드시 확인해야 합니다.

AI에게 넣으면 안 되는 정보 학생 실명, 학번, 생년월일, 연락처, 실제 성적 원자료, 상담 내용, 건강 정보, 가정환경, 징계·생활지도 기록, 특수교육 관련 정보
