2. 핵심 개념

GitHub Pages는 HTML, CSS, JavaScript 같은 정적 파일을 웹사이트로 게시하는 기능입니다. 단순 퀴즈, 체크리스트, 안내 페이지처럼 서버 저장이 없는 수업 도구에 적합합니다.

구분뜻오늘 실습에서의 의미
로컬 실행내 컴퓨터에서 HTML 파일을 직접 열기주소가 보통 file:///로 시작
웹 배포인터넷 주소로 접속할 수 있게 공개주소가 https://...로 시작
정적 사이트서버 계산 없이 파일을 그대로 보여주는 사이트HTML 퀴즈·체크리스트에 적합
백엔드 기능로그인, DB 저장, AI API 호출 같은 서버 작업오늘은 다루지 않음
오늘의 핵심HTML 파일을 만들었다고 자동으로 인터넷 사이트가 되는 것은 아닙니다. GitHub Pages 같은 호스팅 과정을 거쳐야 학생들이 주소로 접속할 수 있습니다.

3. 실습 1 — HTML 파일 준비하기

GitHub Pages에서 첫 화면으로 바로 열리게 하려면 HTML 파일명을 index.html로 준비하는 것이 가장 쉽습니다.

1지난 시간에 만든 HTML 파일 찾기

  • quiz_app_v2.html, checklist.html, rubric_tool.html 같은 파일을 찾습니다.
  • 브라우저로 먼저 열어 정상 작동하는지 확인합니다.
  • 정답·해설·버튼·화면 크기를 확인합니다.

2파일명을 index.html로 바꾸기

quiz_app_v2.html → index.html

오늘은 가장 단순한 방식으로 index.html 하나를 배포합니다.

주의파일명이 index.html.txt가 되지 않도록 확인하세요.

4. 실습 2 — 배포 전 안전 점검

GitHub Pages에 올리면 로컬 파일이 아니라 인터넷 주소가 있는 공개 자료가 됩니다. 배포 전 개인정보·저작권·평가 위험을 반드시 확인합니다.

배포 금지에 가까운 예학생 명단, 실제 평가 결과, 상담 기록, 건강 정보, 생활지도 기록, 내부 업무 자료가 포함된 HTML 파일은 GitHub Pages에 올리지 않습니다.

5. 실습 3 — GitHub 저장소 만들기

저장소(repository)는 웹앱 파일을 올려두는 온라인 폴더라고 생각하면 됩니다.

1GitHub 접속 및 로그인

  • github.com에 접속합니다.
  • 계정이 없다면 회원가입을 합니다.
  • 오른쪽 위 + 버튼을 찾습니다.

2New repository 선택

  • New repository를 클릭합니다.
  • Repository name을 입력합니다.
  • 예: ai-quiz-app, data-checklist, classroom-rubric
  • 오늘은 실습을 위해 Public으로 진행합니다.
  • Create repository를 클릭합니다.
공개 저장소 주의Public 저장소는 코드와 파일이 공개될 수 있습니다. 개인정보나 내부 자료를 넣지 않습니다.

6. 실습 4 — index.html 파일 업로드하기

1Add file → Upload files

  • 저장소 화면에서 Add file을 클릭합니다.
  • Upload files를 선택합니다.
  • index.html 파일을 끌어다 놓습니다.

2Commit changes

  • 파일이 올라간 것을 확인합니다.
  • 아래쪽 Commit changes 버튼을 클릭합니다.
  • 저장소 목록에 index.html이 보이는지 확인합니다.
Commit이란?파일 변경 내용을 GitHub에 저장하는 행동입니다. 오늘은 “저장 버튼” 정도로 이해해도 충분합니다.

7. 실습 5 — GitHub Pages 켜기

이제 저장소에 올린 HTML 파일을 웹사이트로 게시합니다.

1Settings → Pages

  • 저장소 상단의 Settings를 클릭합니다.
  • 왼쪽 메뉴에서 Pages를 클릭합니다.

2Build and deployment 설정

  • Source에서 Deploy from a branch를 선택합니다.
  • Branch에서 main을 선택합니다.
  • 폴더는 /(root)를 선택합니다.
  • Save를 클릭합니다.
UI가 조금 다를 수 있습니다GitHub 화면은 업데이트될 수 있습니다. 핵심은 Settings → Pages → Deploy from a branch → main / root → Save입니다.

8. 실습 6 — 배포 주소 확인하기

잠시 기다리면 GitHub Pages 화면에 웹 주소가 표시됩니다.

https://깃허브아이디.github.io/저장소이름/

예시:

https://leeyy.github.io/ai-quiz-app/
완료 기준내 수업용 HTML 웹앱이 브라우저 주소로 열리면 오늘 실습의 핵심 산출물은 완성입니다.

9. 실습 7 — 수정 후 다시 반영하기

웹앱을 수정하고 싶으면 index.html을 다시 수정하거나 새 파일로 업로드한 뒤 commit하면 됩니다.

방법절차추천 상황
GitHub에서 직접 수정index.html 클릭 → 연필 아이콘 → 수정 → Commit changes작은 문구 수정
수정 파일 다시 업로드Add file → Upload files → 새 index.html 업로드 → Commit changesAI가 만든 수정본 전체 교체
반영 시간commit 후 배포 페이지에 반영되기까지 시간이 조금 걸릴 수 있습니다. 새로고침하거나 잠시 뒤 다시 확인합니다.

10. 수강생용 프롬프트

HTML을 배포하기 전 AI에게 점검시키거나, GitHub Pages용으로 정리할 때 아래 프롬프트를 복사해 사용합니다.

배포 전 HTML 점검 프롬프트

프롬프트 1 — 배포 전 안전 점검
다음 HTML 파일을 GitHub Pages에 배포하기 전에 점검해줘.

점검 기준:
1. 학생 개인정보를 입력받는 부분이 있는지
2. 외부 서버로 데이터를 전송하는 코드가 있는지
3. API 키나 비밀값이 노출되어 있는지
4. 저작권 문제가 생길 수 있는 이미지나 긴 본문이 있는지
5. 학생용 안내문이 충분한지
6. 모바일 화면에서 사용하기 적절한지
7. GitHub Pages에 올려도 되는 정적 HTML 구조인지

결과를 표로 정리하고, 수정이 필요한 부분은 구체적으로 알려줘.

GitHub Pages용 HTML 정리 프롬프트

프롬프트 2 — index.html로 정리
이 HTML 웹앱을 GitHub Pages에 올리기 좋게 정리해줘.

조건:
- 파일명은 index.html로 사용할 예정
- HTML, CSS, JavaScript가 한 파일 안에서 작동
- 외부 API 호출 없음
- 학생 개인정보 입력 없음
- 학생용 안내문 포함
- 모바일에서도 보기 좋게 반응형 적용
- 교사가 나중에 수정할 부분에는 주석 추가

학생 안내 문구 생성 프롬프트

프롬프트 3 — 학생 안내 문구
다음 수업용 웹앱 주소를 학생들에게 안내하는 문구를 만들어줘.

조건:
- 학생이 해야 할 일을 3단계로 안내
- 개인정보를 입력하지 말라는 문구 포함
- 결과는 공식 평가가 아니라 학습 확인용이라는 점 포함
- 고등학생이 이해하기 쉬운 말투

웹앱 주소:

오늘의 핵심 문장

HTML 수업 도구를 배포한다는 것은 “파일을 공유하는 것”을 넘어, 인터넷 주소가 있는 공개 자료로 만드는 일입니다. 배포 전에는 반드시 개인정보, 저작권, 정답 오류, 평가 오해 가능성을 점검해야 합니다.

인쇄가 필요하면 브라우저에서 Ctrl+P 또는 ⌘+P를 누르세요. 복사 버튼은 인쇄 시 자동으로 숨겨집니다.
